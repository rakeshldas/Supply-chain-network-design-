# 10. Creating Factory-to-Depot Flow Variables

```python
flow_fd = model.addVars(
    factories, depots,
    lb=0,
    vtype=GRB.CONTINUOUS,
    name="Factory_to_Depot_Flow"
)
```

This creates decision variables for quantity shipped from factories to depots.

The variable is called:

```text
flow_fd
```

It means:

```text
flow_fd[f, d] = quantity shipped from factory f to depot d
```

Example:

```text
flow_fd["Factory_1", "Depot_2"]
```

means:

```text
quantity shipped from Factory_1 to Depot_2
```

Explanation of the arguments:

```text
factories, depots
```

This creates one variable for every factory-depot pair.

Since there are 2 factories and 3 depots, the model creates:

```text
2 × 3 = 6 factory-to-depot flow variables
```

---

```text
lb=0
```

This means lower bound is 0.

So shipment quantity cannot be negative.

---

```text
vtype=GRB.CONTINUOUS
```

This means the variable is continuous.

So the flow can technically be 10.5, 25.75, or any non-negative value.

In real industrial cases, if the unit must be whole cartons or trucks, we can change it to integer.

---

```text
name="Factory_to_Depot_Flow"
```

This gives a readable name to the variable inside Gurobi.

# 13. Objective Function

```python
model.setObjective(
```

This tells Gurobi that we are now defining the objective function.

The objective function is the main goal of the model.

Here, our goal is:

```text
Minimize total supply chain cost
```

---

```python
    gp.quicksum(cost_factory_to_depot[f, d] * flow_fd[f, d]
                for f in factories for d in depots)
```

This is the first part of the objective function.

It calculates total factory-to-depot transportation cost.

For every factory and every depot:

```text
cost per unit from factory to depot × quantity shipped from factory to depot
```

Example:

```text
If Factory_1 ships 100 units to Depot_1
and the cost per unit is 4,
then cost = 4 × 100 = 400
```

`gp.quicksum()` is Gurobi’s efficient way to add many terms.

---

```python
    +
    gp.quicksum(cost_depot_to_customer[d, c] * flow_dc[d, c]
                for d in depots for c in customers)
```

This is the second part of the objective function.

It calculates total depot-to-customer transportation cost.

For every depot and every customer:

```text
cost per unit from depot to customer × quantity shipped from depot to customer
```

Example:

```text
If Depot_2 ships 220 units to Customer_2
and the cost per unit is 3,
then cost = 3 × 220 = 660
```

---

```python
    +
    gp.quicksum(fixed_depot_cost[d] * open_depot[d]
                for d in depots),
```

This is the third part of the objective function.

It calculates total fixed cost for opening depots.

For every depot:

```text
fixed opening cost × depot opening decision
```

Example:

```text
If Depot_1 is opened:
open_depot["Depot_1"] = 1
Fixed cost = 1200 × 1 = 1200
```

If Depot_1 is closed:

```text
open_depot["Depot_1"] = 0
Fixed cost = 1200 × 0 = 0
```

So the fixed cost is paid only when the depot is opened.

---

```python
    GRB.MINIMIZE
)
```

This tells Gurobi that the objective is to minimize the total cost.

So Gurobi will search for the lowest-cost feasible supply chain design.

# 14. Factory Capacity Constraints

```python
for f in factories:
```

This loop goes through each factory one by one.

First it handles `Factory_1`, then `Factory_2`.

---

```python
    model.addConstr(
```

This tells Gurobi that we are adding a constraint.

A constraint is a rule that the solution must follow.

---

```python
        gp.quicksum(flow_fd[f, d] for d in depots) <= factory_capacity[f],
```

This is the factory capacity constraint.

It means:

```text
Total quantity shipped from factory f to all depots
must be less than or equal to
capacity of factory f
```

For Factory_1, this becomes:

```text
flow from Factory_1 to Depot_1
+ flow from Factory_1 to Depot_2
+ flow from Factory_1 to Depot_3
<= 500
```

For Factory_2, this becomes:

```text
flow from Factory_2 to Depot_1
+ flow from Factory_2 to Depot_2
+ flow from Factory_2 to Depot_3
<= 450
```

---

```python
        name=f"Factory_Capacity_{f}"
    )
```

This gives a name to the constraint.

For example:

```text
Factory_Capacity_Factory_1
Factory_Capacity_Factory_2
```

This helps when debugging or checking the model.

# 15. Depot Flow Balance Constraints

```python
for d in depots:
```

This loop goes through each depot one by one.

It handles:

```text
Depot_1
Depot_2
Depot_3
```

---

```python
    model.addConstr(
```

This adds a new constraint to the model.

---

```python
        gp.quicksum(flow_fd[f, d] for f in factories)
        ==
        gp.quicksum(flow_dc[d, c] for c in customers),
```

This is the depot flow balance constraint.

It means:

```text
Total quantity entering a depot from all factories
must be equal to
total quantity leaving that depot to all customers
```

For Depot_1, this means:

```text
Factory_1 to Depot_1 flow
+ Factory_2 to Depot_1 flow
=
Depot_1 to Customer_1 flow
+ Depot_1 to Customer_2 flow
+ Depot_1 to Customer_3 flow
+ Depot_1 to Customer_4 flow
```

This ensures that the depot does not create or lose products.

Whatever enters the depot must leave the depot.

---

```python
        name=f"Depot_Flow_Balance_{d}"
    )
```

This gives a name to each depot flow balance constraint.

Examples:

```text
Depot_Flow_Balance_Depot_1
Depot_Flow_Balance_Depot_2
Depot_Flow_Balance_Depot_3
```

# 16. Depot Capacity Constraints Linked with Depot Opening

```python
for d in depots:
```

This loop again goes through each depot.

---

```python
    model.addConstr(
```

This adds another constraint.

---

```python
        gp.quicksum(flow_dc[d, c] for c in customers)
        <= depot_capacity[d] * open_depot[d],
```

This is one of the most important constraints in the model.

It links depot flow with depot opening decision.

It means:

```text
Total quantity shipped from depot d to all customers
must be less than or equal to
depot capacity × open depot decision
```

Now understand the logic carefully.

If Depot_1 is closed:

```text
open_depot["Depot_1"] = 0
```

Then:

```text
Total shipment from Depot_1 <= 400 × 0
```

So:

```text
Total shipment from Depot_1 <= 0
```

This means Depot_1 cannot ship anything.

If Depot_1 is opened:

```text
open_depot["Depot_1"] = 1
```

Then:

```text
Total shipment from Depot_1 <= 400 × 1
```

So:

```text
Total shipment from Depot_1 <= 400
```

This means Depot_1 can ship up to 400 units.

This is called a linking constraint or activation constraint.

---

```python
        name=f"Depot_Capacity_{d}"
    )
```

This gives a name to the depot capacity constraint.

Examples:

```text
Depot_Capacity_Depot_1
Depot_Capacity_Depot_2
Depot_Capacity_Depot_3
```

# 26. Simple Explanation of the Whole Code

This code solves a supply chain network design problem.

The company has:

```text
2 factories
3 possible depots
4 customers
```

The model decides:

```text
1. how much to ship from each factory to each depot
2. how much to ship from each depot to each customer
3. which depots to open
```

The objective is:

```text
Minimize total supply chain cost
```

Total cost includes:

```text
factory-to-depot shipping cost
+ depot-to-customer shipping cost
+ fixed cost of opening depots
```

The model follows these rules:

```text
Factories cannot ship more than their capacity.
Depots cannot be used unless opened.
Open depots cannot exceed their capacity.
Whatever enters a depot must leave the depot.
Every customer demand must be fully satisfied.
```
